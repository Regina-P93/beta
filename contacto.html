Formularios HTML

Cual es su funcion?

Son herramientas cuya funcion principal es permitir la interaccion (intercambio de datos) entre el cliente y el servidor, debido a que es la unica parte dentro de la pagina donde el usuario puede modificar directamente el contenido. Para poder utilizarlos debemos colocar la etiqueta < form dentro de la pagina, ademas debemos definir algunos atributos y elementos que se mencionan mas adelante. Solamente se enviara lo que se encuentre dentro de dicha etiqueta y ninguna otra cosa

  • Buscador - Consulta Query
  • Registro - Inscripcion
  • Acceso - Inicio de sesión
  • Contacto - Consulta Web

Como funcionan

A la etiqueta anterior , debemos colocarlos por lo menos dos atributos; por un lado action ="..." que indica lo que debe suceder al momento de enviar el formulario, hoy en dia se suele utilizar un script en lenguaje de sevidor (ej: php) para garantizar la correcta manipulacion de los datos. Por otro lado esta el atributo method="..." que define como se realizara el envio de los datos y cual es prioritario (cliente/servidor), segun los valores GET y POST.

  • GET: metodo menos seguro, donde los datos siempre estan visibles en la barra de direcciones y es mas rapido debido a que la prioridad de respuesta es del servidor. Los datos se envian en tipo plano no estan encriptados y son más propensos a robo de datos
  • POST: metodo más seguro donde los datos se encriptan antes de realizarse el envio pero es mucho mas lento debido al proceso que esto implica y la prioridad esta en el los datos enviados por el usuario. No hay garantia de que no se roben los datos pero es más dificil

Que elementos poseen

Los formularios poseen dos tipos de elementos, por un lado están las etiquetas(LABEL) que funcionan a modo de rotulos que indican al usuario que datos debe ingresar / seleccionar dentro del formulario, ademas facilitan la accesibilidad (el navegador lee lo que dicen estas etiquetas); por otro lados estan los campos(INPUT / TEXTAREA / SELECT) cuya funcion es permitir el ingreso / seleccion de datos, poseen dos atributos fundamentales que son type="..." para indicar el tipo de dato a ingresar y value="..." que representa el dato ingresado.

TABLAS HTML

¿Cuando utilizarlas?

Son elementos que permiten organizar la informacion de manera tabular y deben utilizarse cuando los datos que queremos representar en pantalla deben respetar cierta estructura. Al igual que en cualquier otro ambito, poseen filas, columnas y celdas, pero la manera de crearlas es mediante el modelo de cajas, es decir, utilizando etiquetas contenedoras y etiquetas de contenido.

¿Poseen estructura semantica?

A partir del nuevo estandar HTML5, las tablas deben tener definida una estructura semantica, para facilitar a los motores de busqueda el indexado y analisis de las mismas. Una caracteristicas a destacar es que las nuevas etiquetas introducidas, reemplazan a las filas y celdas en la mayoria de los casos; las mismas se colocan dentro del contenedor de la tabla <table respetando cierto orden, como se observa a continuacion:

  • THEAD: Cabecera de tabla, se utiliza para la definicion de los titulos y en su interior posee celdas de titulo (TH). Es obligatorio.
  • TBODY: Cuerpo de tabla, es la unica parte que mantiene la estructura clasica, aunque en ciertas situaciones también puede poseer titulos. Es obligatorio.
  • TFOOT: Pie de tabla, complementario al contenido principal de la misma, suele poseer totales, promedios u otros valores estadisticos. Es optativo

Contenido Multimedia

¿Contenido embebido?

Antiguamente si necesitabamos incorporar este tipo de contenido, era necesaria la instalacion de complementos o plugins de terceros, aunque podiamos utilizar etiquetas como < iframe para incorporar material externo de manera asincrona a nuestro sitio. Hoy se utiliza para aprovechar una caracteristica de la Web que se conoce como contenido embebido o compartido, mientras que para el contenido multimedia es mas comun utilizar las etiquetas que se veran mas adelante.

¿Sonidos en la Web?

Como se menciono anteriormente, hoy en día ya no es necesario contar con herramientas de terceros para la reproduccion de contenido multimedia. Con la etiqueta <audio podemos utilizar la API de reproduccion incorporada al navegador, que es compatible con los formatos .mp3 - .wav - .ogg, aunque tambien soporta formatos de pelicula como los que se mencionan mas adelante.

¿Multimedia o video sin plugins?

Como se menciono, esta etiqueta surge para solventar el inconveniente de reproduccion de contenido multimedia, la etiqueta para esto es <video y admite los formatos .mp4 - .webm -.ogv. A diferencia del anterior , no es necesario utilizar el atributo controls e incluso puede utilizar atributos como autoplay y muted que deben estar juntos para funcionar.
